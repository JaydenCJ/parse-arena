## [0.1.0] - 2026-07-08

### Added

- Dataset manifest format (YAML) with per-document ground truth (text blocks,
  tables with colspan/rowspan merged cells, key/value fields, vertical flag)
  and track labels.
- Built-in synthetic dataset: 11 documents across 5 tracks (`en-text`,
  `en-table`, `en-form`, `ja-vertical`, `ja-receipt`), including merged-cell
  tables (HTML and PDF), a born-digital form, vertical Japanese text and
  Japanese receipts; PDF fixtures are generated by
  `scripts/gen_pdf_fixture.py`.
- Parser adapter framework: abstract base class, registry, availability
  reporting; built-in adapters `plaintext`, `html-stdlib` (merged-cell grid
  extraction, vertical-rl layout aware), `pypdf`, `mock-oracle` (calibration
  baseline), plus optional `unstructured` and `markitdown` adapters that skip
  gracefully when the dependency is missing.
- Metrics: CER/WER, simplified TEDS table similarity on span-expanded grids,
  reading-order Kendall tau, Japanese vertical-order accuracy, field recall;
  documented normalization to a single per-document score.
- CLI: `parse-arena run` (results JSON), `parse-arena report` (static
  leaderboard site with track switching and reproduce command),
  `parse-arena serve` (loopback preview server), `parse-arena router`
  (best-parser-per-track routing YAML), `parse-arena parsers`.
- `docker-compose.yml`: one-command deployment — a harness service benchmarks
  the dataset and publishes the static site into a named volume served by a
  pinned nginx with healthcheck.
- Test suite (pytest) and `scripts/smoke.sh` end-to-end smoke script.
